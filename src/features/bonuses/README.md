# Модуль bonuses

Модуль `bonuses` відповідає за управління бонусними рахунками та транзакціями клієнтів.

## Структура папки

- `model.py` — SQLAlchemy моделі для бонусних рахунків (`BonusAccount`) та транзакцій (`Transaction`).
- `schemas.py` — Pydantic-схеми для валідації та серіалізації даних бонусних рахунків і транзакцій.
- `controller.py` — Контролери для CRUD-операцій над бонусними рахунками та транзакціями (generic CRUD через фабрику).
- `routes.py` — FastAPI-роутери для API бонусних рахунків і транзакцій, з підключенням авторизації та CRUD-операцій.
- `__init__.py` — ініціалізація модуля.

## Опис

Модуль дозволяє:

- Створювати, оновлювати та переглядати бонусні рахунки клієнтів.
- Створювати та переглядати транзакції по бонусах (нарахування/списання).
- Використовує generic CRUD-контролери та роутери для швидкої розробки.
- Підтримує авторизацію через JWT.

### Основні моделі

- `BonusAccount`: зберігає інформацію про бонусний рахунок клієнта (баланс, клієнт, дата оновлення).
- `Transaction`: зберігає інформацію про транзакції (нарахування/списання бонусів).

### Основні ендпоінти

- `/bonus-accounts` — CRUD для бонусних рахунків (тільки list, можна розширити).
- `/bonus-transactions` — CRUD для транзакцій (тільки list, можна розширити).

### Додатково

- Всі ендпоінти захищені авторизацією.
- Можливість розширення під бізнес-логіку.
